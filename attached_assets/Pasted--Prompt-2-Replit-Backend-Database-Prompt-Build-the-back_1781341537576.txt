# Prompt 2 — Replit Backend + Database Prompt

Build the backend, database, authentication, authorization, and persistence layer for **Japan Nepali Shop SaaS**.

The current project already has a polished frontend prototype using Next.js, React, Tailwind, and shadcn/ui. Now convert it into a real production-ready multi-tenant SaaS backend.

## Main Goal

Replace all mock data with real database-backed functionality using PostgreSQL.

The platform is a multi-vendor marketplace SaaS for Nepali shops in Japan.

There are four main roles:

1. Customer
2. Merchant
3. Admin
4. Platform Super Admin

## Multi-Tenant Rules

Every merchant/shop/order/product must belong to a tenant.

Use this structure:

```text
Tenant
 ├── Users
 ├── Shops
 ├── Products
 ├── Orders
 ├── Categories
 ├── Settings
 └── Subscription
```

Do not allow data leakage between tenants.

All queries must be scoped by `tenantId`, except Platform Super Admin.

## File Storage

Replace base64 images with S3 storage.

S3 path structure:

```text
japannepalishop/
└── tenants/
    └── {tenantCode}/
        ├── _info.json
        ├── shops/
        ├── products/
        ├── banners/
        ├── categories/
        ├── exports/
        └── trash/
```

Rules:

* Use tenantCode, not tenantId, in S3 folder path
* Creating tenant should create base folder structure
* Product images go inside products folder
* Shop logos/banners go inside shops or banners folder
* Deleting files should move them to trash first
* Do not break existing image URLs
